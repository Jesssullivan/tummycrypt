## DB Architecture Notes
## ============================================================

# Two separate database files with independent connection managers:
#
# 1. SyncTrackingDB (sync9.sync_tracking_service.internals.db)
#    - SyncTrackingDBConnection
#    - SyncTrackingDBService
#    - Contains: SyncTrackingTable
#    - Purpose: Stores the entire local<->remote node tree
#
# 2. PropertyDB (odrive_app.services.db)
#    - PropertyDBConnection
#    - PropertyDBService
#    - Contains: PropertyTable, SyncModeTable, StickySyncTable,
#                EncryptionEntryTable, BackupJobTable, ProSyncFolderTable
#    - Purpose: Application settings and configuration
#
# 3. IntegrationCacheDB (integrations)
#    - IntegrationCacheDBConnection
#    - IntegrationCacheDBService
#    - Contains: IntegrationCacheTable
#    - Purpose: Cached remote listing data
#
# All use APSW (Another Python SQLite Wrapper) for thread-safe access.
# DBConnectionManager handles per-thread connection pooling.
