## Key Design Patterns

1. **Adapter Pattern**: Each cloud provider has a `Sync9Adapter` (integration layer)
   and a `*Service` (API-specific implementation). `SyncAdapterFactory` selects the right one.

2. **Encryption as Integration**: Encryption is implemented as a Sync9Adapter
   (`EncryptionSync9Adapter`), not as a transport layer. This means encryption
   is a "virtual storage provider" that wraps another provider.

3. **Sync9 Engine**: The core sync engine is versioned ("sync9"), with
   `SyncTrackingService` maintaining per-node sync state via `SyncTrackingNode`.

4. **Refresh/Merge Pipeline**: The refresh_job controller has a multi-stage
   pipeline: Compare -> GroupByPosition -> MergeFiles -> Operations,
   with separate LocalAddFolder/RemoteAddFolder handlers.

5. **Event-Driven**: Dual event system with LocalEventController (FS events)
   and RemoteEventController (cloud-side changes).

6. **Protocol Server**: Agent exposes a ProtocolServer for CLI/UI communication,
   with ProtocolCommands defining the command set. The CLI binary (`odrive`)
   connects to this server.
