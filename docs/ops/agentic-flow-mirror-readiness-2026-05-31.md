## Deny Rules

Hard deny before any manifest or staging copy:

- Credential/auth/env names: `auth.json`, `.credentials.json`,
  `mcp-auth.json`, `mcp-needs-auth-cache.json`, `session-env`, `.env`,
  `.envrc`, `.env.*`, `*.env`, `*.env.*`, `.netrc`, `.pgpass`.
- Secret stores: `.ssh`, `.gnupg`, `sops-nix`, `nix/secrets`.
- Live database files: `*.sqlite`, `*.sqlite3`, `*.db`, `*.sqlite-wal`,
  `*.sqlite-shm`, `*.db-wal`, `*.db-shm`, `*-wal`, `*-shm`.
- Runtime/caches: `*.log`, `.cache`, `node_modules`, `.tmp`, `log/`, `logs/`,
  plugin caches, package caches, `.claude/worktrees`, `.crush/logs`, generated
  build outputs.
- Repo generated outputs: `target`, `node_modules`, `.svelte-kit`, `build`,
  `.direnv`, `.venv`, `.cache`, `.artifacts` unless a repo-specific artifact
  row explicitly opts in.

These denies mirror the enforced TCFS `Blacklist` direction. If a file is
denied here but desired later, it must move through an explicit snapshot or
artifact lane rather than bypassing the deny set.

## Snapshot Rules

SQLite/DB files are allowed only through a snapshot product:

1. Confirm the source DB exists and record size, mtime, and journal mode.
2. Use `sqlite3 SOURCE ".backup SNAPSHOT"`; do not copy the source DB, WAL, or
   SHM file.
3. Run `sqlite3 SNAPSHOT "PRAGMA integrity_check;"` and require `ok`.
4. Hash the snapshot and record source path, snapshot path, byte size, hash,
   journal mode, and timestamp in the mirror manifest.
5. Copy only the snapshot output into staging, never the live source sidecars.

Append-style JSONL files are lower risk but still need active-writer handling:
record size and hash before and after a short bounded interval; if they change,
mark them `defer-active-writer` rather than staging them.

## First Manifest Shape

The prepare-only manifest is generated by:

```bash
scripts/agentic-flow-mirror-inventory.py SOURCE... --out-dir /tmp/tcfs-agentic-flow-manifest
```

It writes `manifest.jsonl` and `summary.json`, does not follow symlinks, does
not copy data, and never hashes denied or snapshot-required files. The manifest
contains one row per candidate:

```text
source_path
surface_class        # agent_config | transcript | sqlite_snapshot | repo | tmp_candidate
decision             # allow | deny | snapshot | defer-active-writer | future-ttl
reason
size_bytes
mtime_utc
hash_sha256          # for files/snapshots
repo_branch
repo_dirty_summary
remote_summary
snapshot_source      # for sqlite_snapshot rows
```

No row marked `deny`, `defer-active-writer`, or `future-ttl` may enter a staging
copy. No staging copy should be sent to `honey` until `TIN-1417` and `TIN-1736`
are green.
